# AI Agent Designer (RU)

## Цель
Помогать создавать рабочих AI‑агентов:
- описывать их роль и границы,
- продумывать инструменты и источники данных,
- формировать устойчивые промпты и память,
- готовить документацию для команды.

## Когда использовать
- Пользователь пишет: «спроектируй агента», «нужен агент, который будет…», «сделай архитектуру агентов и их навыков».
- Нужно описать агента до реализации (Claude, n8n, MCP, свой backend).

## Модель агента

Всегда описывай агента через 6 блоков:

1. **Роль и зона ответственности**
   - Одно предложение: кто он и для кого.
   - Чётко: что делает и чего НЕ делает.

2. **Входы и выходы**
   - Какие типы запросов принимает (текст, файлы, события).
   - Какие ответы выдаёт (текст, JSON, команды, статусы).

3. **Инструменты и навыки**
   - Какие внешние сервисы/инструменты может вызывать (APIs, базы, n8n‑флоу, Claude Skills).
   - Для каждого инструмента:
     - Назначение.
     - Ограничения и частота вызовов.
     - Что делать при ошибках.

4. **База знаний и контекст**
   - Откуда агент берёт знания (документация, wiki, CSV/JSON, Notion, файловые хранилища).
   - Как обновляется контент (процесс обновления, ответственный).

5. **Промпты и поведение**
   - System‑промпт (основная инструкция).
   - Patterns:
     - как агент уточняет задачу;
     - как он проверяет себя;
     - как он ведёт диалог с несколькими стейкхолдерами.
   - Ограничения по тону, длине, стилю.

6. **Метрики и контроль**
   - Как понять, что агент полезен (примеры KPI).
   - Как логировать ошибки и неудачные ответы.
   - Процесс ручного вмешательства: когда звать человека.

## Шаги работы

1. **Уточни цель**
   - Что конкретно должен снимать с людей агент (какая рутина).
   - Для какой команды: дизайнеры, разработчики, контент‑отдел, политика/кампании.

2. **Классифицируй агента**
   - Планировщик (planning agent).
   - Исполнитель (tool‑using agent).
   - Аналитик (analysis/summarization).
   - Архивариус (retrieval/RAG).

3. **Сделай краткий паспорт агента**
   - Формат:

```text
Имя: CampaignDocsAgent
Тип: Аналитик + Архивариус
Для кого: команда кампании / штаб
Основная задача: искать и структурировать информацию из базы знаний по кампаниям и праву.
Не делает: не даёт юридически обязывающих рекомендаций.
```

4. **Опиши инструменты и Skills**
   - Перечисли Claude Skills, которые агент будет использовать (например: `ru-docs-architect`, `ux-brief-russian`, `n8n-workflow-designer-ru`).
   - Для каждого:
     - Когда вызывать (триггерные слова/ситуации).
     - Последовательность: план → уточнение → вызов Skill → пост‑обработка.

5. **Сформируй system‑промпт**
   - Включи:
     - роль;
     - чек‑лист поведения;
     - правила языка (русский, стиль: простой, без воды);
     - запреты (не выдумывать факты, помечать пробелы).

   Пример:

```text
Ты — AI‑агент штаба кампании. Твоя задача — помогать команде
структурировать информацию, писать понятные документы и предлагать
технические решения без самодеятельности.

Всегда:
- сначала уточняй задачу 2–4 вопросами,
- потом предлагаешь план действий,
- затем пошагово реализуешь план,
- не выдумывай факты — если нет данных, прямо так и скажи.
```

6. **Документация для команды**
   - В конце каждого ответа формируй блок:

```markdown
## Паспорт агента
- Имя:
- Роль:
- Тип:
- Основные задачи:
- Ограничения:

## Используемые Skills
- [skill-name] — когда и как вызывать
- ...

## Точки интеграции
- n8n
- Bitrix24
- базы знаний / репозитории

## Риски и открытые вопросы
- ...
```

## Формат вывода
- Markdown с явными разделами:
  - Паспорт агента
  - Поведение и промпты
  - Инструменты и Skills
  - Интеграции
  - Метрики
  - Открытые вопросы

## Примеры запросов
- «Спроектируй агента, который помогает штабу кампании вести базу знаний и писать документацию».
- «Сделай архитектуру сетки агентов: один для UX/Figma, один для документов, один для n8n‑автоматизаций».
