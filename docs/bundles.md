# Skill Bundles (наборы)

Готовые комбинации навыков для типичных задач. Используйте несколько навыков вместе для комплексных сценариев.

## 1. Новый сайт кампании / партии

**Рекомендуемый набор:**
- `ux-brief-russian` — собрать бриф и «Понимание задачи».
- `political-campaign-site-planner` — карта сайта и контент‑план.
- `figma-ux-flow-planner-ru` — структура файлов и фреймов в Figma.
- `ux-spec-for-dev-russian` — UX/UI‑спека для разработки.
- `ru-content-ilyahov-style` — тексты для страниц.

**Порядок работы:**
1. Собрать бриф из переписки с клиентом (`ux-brief-russian`)
2. Спланировать структуру сайта (`political-campaign-site-planner`)
3. Разложить по экранам и фреймам в Figma (`figma-ux-flow-planner-ru`)
4. Сделать спецификацию для разработчиков (`ux-spec-for-dev-russian`)
5. Написать тексты для страниц (`ru-content-ilyahov-style`)

## 2. Автоматизация штаба (n8n + боты + CRM)

**Рекомендуемый набор:**
- `n8n-workflow-designer-ru` — проектирование автоматизаций.
- `telegram-bot-architect-ru` — бот для заявок/уведомлений.
- `server-admin-consultant-ru` — план развёртывания и безопасной эксплуатации.
- `ru-docs-architect` — документация по всем сценариям.

**Что автоматизировать:**
- Сбор заявок из Telegram в CRM
- Уведомления о событиях в рабочие чаты
- Синхронизация контактов между системами
- Мониторинг и отчёты

## 3. Внутренняя база знаний + RAG‑агент

**Рекомендуемый набор:**
- `ru-docs-architect` — оформить базу знаний и структуру документов.
- `vector-db-architect-ru` — спроектировать коллекции векторной БД и пайплайн RAG.
- `mcp-architect-ru` — описать MCP‑сервер, дающий доступ к векторной БД.
- `ai-agent-designer-ru` — спроектировать агента для работы с базой знаний.

**Архитектура агента:**
- **Вход:** вопрос от штаба/пользователя
- **Действие:** вызывает MCP‑инструмент поиска по векторной БД
- **Выход:** формирует ответ с указанием источников

## 4. Пакет «Дизайн → Код → Документация»

**Рекомендуемый набор:**
- `ux-brief-russian` — формулируем задачу.
- `figma-ux-flow-planner-ru` — структура Figma.
- `ux-spec-for-dev-russian` — спека для фронта/бэка.
- `ru-docs-architect` — README, ТЗ, гайды для команды.
- `ai-agent-designer-ru` — агент для поддержки документации в актуальном состоянии.

**Результат:** полный цикл от идеи до задокументированного продукта

## 5. Техническая платформа кампании

**Рекомендуемый набор:**
- `server-admin-consultant-ru` — архитектура серверов, деплой, бэкапы.
- `n8n-workflow-designer-ru` — интеграции (Telegram, Bitrix24, почта, мониторинг).
- `telegram-bot-architect-ru` — бот для пользователей и внутренней команды.
- `mcp-architect-ru` + `vector-db-architect-ru` — доступ к данным и базе знаний из Claude.
- `ai-agent-designer-ru` — агенты для разных ролей: контент, аналитика, техподдержка.

**Компоненты платформы:**
```
┌─────────────────────────────────────────────────┐
│  Telegram Bot  │  n8n Workflows  │  Web Panel   │
├─────────────────────────────────────────────────┤
│            MCP Server + Vector DB               │
├─────────────────────────────────────────────────┤
│         AI Agents (контент, аналитика, supp)    │
└─────────────────────────────────────────────────┘
```

## 6. Быстрый старт для нового проекта

**Минимальный набор:**
- `ux-brief-russian` — понять задачу
- `ru-docs-architect` — оформить README и ТЗ

**Когда использовать:**
- Новый репозиторий или проект
- Нужно быстро структурировать задачу
- Базовая документация для команды

## 7. Кросс-функциональная команда

**Набор для агентного подхода:**
- `ai-agent-designer-ru` — спроектировать агентов для каждой роли
- `mcp-architect-ru` — интеграции между агентами
- `vector-db-architect-ru` — общая база знаний

**Пример ролей агентов:**
- **UX Agent** — использует `ux-brief-russian`, `figma-ux-flow-planner-ru`
- **Dev Agent** — использует `ux-spec-for-dev-russian`, `server-admin-consultant-ru`
- **Docs Agent** — использует `ru-docs-architect`, `ru-content-ilyahov-style`
- **Automation Agent** — использует `n8n-workflow-designer-ru`, `telegram-bot-architect-ru`
